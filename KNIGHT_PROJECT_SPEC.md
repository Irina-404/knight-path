# Knight’s Path — описание проекта

> Игра создана по мотивам главы 10 («Стратегическая фигура») книги «Миригор».
> Встраивается в сайт книги как отдельный интерактивный компонент.
> Язык интерфейса: английский (V1), с возможностью перевода в будущем.

---

## Идентификаторы

| Назначение | Значение |
|---|---|
| Название игры | **Knight’s Path** |
| Краткое описание | *Help the knight visit every square exactly once.* |
| Папка / slug | `knight-path` |

---

## Суть задачи

Игрок управляет шахматным конём на квадратной доске.

Конь должен:
- посетить каждую клетку доски ровно один раз;
- двигаться только ходом шахматного коня.

Игра вдохновлена классической задачей Knight’s Tour, но основной акцент сделан не на шахматах, а на стратегии выбора следующего хода.

Главная идея игры — показать разницу между хаотичным выбором и стратегическим планированием: игроку нужно не просто прыгать по доске, а думать, какие клетки могут стать узкими местами.

---

## Что может делать игрок

- Выбрать размер доски: от 5×5 до 8×8.
- Включить или выключить подсказки Варнсдорфа (числа продолжений на допустимых клетках).
- Начать маршрут со стартовой клетки `(0, 0)` (левый нижний угол доски).
- Кликать по допустимым клеткам для следующего хода.
- Откатывать ходы назад (`Undo`).
- Сдаться и посмотреть один из возможных правильных маршрутов (`Give up?`).

---

## Правила игры

Показываются на экране настроек:

```text
• Move the knight using normal knight moves.
• Visit every square exactly once.
• Try not to trap yourself.
```

Подсказка про числа продолжений раньше дублировалась здесь как «Numbers show how many continuations each move has.» — теперь эта строка живёт под переключателем `Continuation hints` и в правилах не дублируется.

## Размер доски

| Размер | Доступность |
|--------|-------------|
| 5×5 | по умолчанию |
| 6×6 | доступно |
| 7×7 | доступно |
| 8×8 | доступно |

Настройка сохраняется в `localStorage`.

---

## Стартовая клетка

Маршрут всегда начинается из угловой клетки `(0, 0)` (левый нижний угол доски). Из этой клетки на любом размере 5–8 существует хотя бы один открытый Knight's Tour, поэтому задача всегда разрешима.

Расширение на будущее: случайный выбор одной из клеток, для которой существует валидный маршрут. В V1 не реализуется.

---

## Логика победы

Игрок побеждает, если все клетки доски посещены ровно один раз. Любой полный маршрут — победа; финишная клетка может быть любой.

Концепция «идеального прохождения с финишем у выхода» в V1 не используется. Из главы книги оно имело смысл только для доски 5×5, а на остальных размерах создавало бы непредсказуемые требования к чётности и могло наказывать за фактически решённую задачу.

---

## Логика поражения

Поражение наступает, если:
- существуют непосещённые клетки;
- но у коня больше нет допустимых ходов.

Поражение подаётся не как жёсткое наказание, а как сюжетный тупик: конь застрял, как в главе.

---

## Подсказки ходов

В игре есть **два режима**, выбираемых настройкой `Continuation hints: On / Off` (по умолчанию `On`):

**Hints ON — режим с подсказками.** На каждой допустимой клетке следующего хода:
- мягкое бирюзовое мерцание (того же ритма, что пульсация золотой кнопки `Start game`);
- число продолжений (эвристика Варнсдорфа) поверх мерцания;
- если число равно `0`, мерцание окрашивается в красный — клетка-ловушка.

**Hints OFF — чистая доска.** Допустимые клетки никак не выделены. Игрок видит только саму доску, посещённые клетки с номерами шагов и коня. Никаких бирюзовых, красных подсветок и чисел нет.

Эвристика Варнсдорфа: для каждой клетки-кандидата `C` число продолжений равно количеству допустимых ходов коня **из клетки `C` после того, как `C` будет посещена** (то есть степень `C` в графе ещё непосещённых клеток с учётом, что сама `C` уже посещена).

Пример (в режиме hints ON):
- `5` — из клетки `C` есть 5 возможных продолжений;
- `0` — клетка `C` ведёт в тупик: попав в неё, конь не сможет двинуться дальше.

Это ключевая механика игры. Она повторяет идею из главы: важно не только видеть ближайший ход, но и оценивать, сколько возможностей останется после него.

Режим OFF существует для тех, кто хочет пройти задачу «честно», без эвристической помощи: и без чисел, и без подсветки кандидатов. Игрок без подсказок узнаёт о тупике, только попав в него.

### Когда что показывается на клетке

В каждый момент на одной клетке отображается **либо** число продолжений, **либо** номер шага — никогда оба одновременно:

- клетка с конём: рисуется фигура коня, без числа;
- допустимые клетки следующего хода (вокруг коня):
  - при `showCounts === true` — бирюзовое мерцание + число продолжений (или красное мерцание + `0` для тупиков);
  - при `showCounts === false` — нейтральный фон, как у всех непосещённых клеток;
- посещённые клетки (где конь побывал и ушёл): затемнены, на каждой — номер шага;
- остальные клетки (непосещённые, недоступные с текущей позиции): нейтральный фон.

После каждого хода: бывшая клетка коня переключается из «текущей» в «посещённую с номером», новая клетка коня — в «текущую без числа», подсветки и числа продолжений пересчитываются для новой позиции (или просто исчезают, если подсказки выключены).

## Undo

Игрок может откатить маршрут назад.

Кнопка:
- `Undo`

Поведение:
- **один клик = один ход**: каждое нажатие откатывает ровно один последний ход;
- кнопку можно нажимать многократно подряд — каждое следующее нажатие откатывает ещё один ход;
- лимита на глубину отката нет: можно откатить весь маршрут вплоть до стартовой клетки;
- кнопка деактивируется (серый стиль, клики игнорируются), когда конь уже стоит на стартовой клетке (`path.length < 2`) — откатывать больше нечего;
- маршрут и доступные ходы пересчитываются полностью после каждого отката.

Undo доступен только во время активной игры (фаза `playing`). Хоткеи: `U` или `Ctrl+Z`.

Undo здесь не считается вспомогательным костылём. Это часть нормальной игровой механики, потому что задача длинная, а ошибка может проявиться через много ходов. Без отката игра быстро стала бы раздражающей, особенно на досках 6×6, 7×7 и 8×8.

---

## История ходов

Так как доска квадратная, а canvas имеет пропорцию 4:3, справа от доски можно использовать вертикальную сервисную колонку.

В этой колонке можно отображать историю маршрута:
- номер хода;
- координату клетки;
- текущий выбранный ход.

Игрок может выбрать ход в истории и откатиться до него.

Минимальная версия:
- только кнопка `Undo`, откат на один ход.

Расширенная версия:
- список ходов справа;
- клик по ходу выбирает точку отката;
- кнопка `Undo to selected` или аналогичная выполняет откат.

Рекомендуемый путь реализации для V1:
- сначала сделать простой `Undo` на один ход;
- если останется время, добавить историю ходов как расширенный инструмент.

### Финальный экран

После завершения игры:
- маршрут игрока остаётся на доске;
- показывается результат;
- если игрок сдался — поверх стираются ходы игрока и анимированно прорисовывается один из валидных маршрутов;
- доступны:
  - `Settings & Rules` — открывает экран настроек;
  - `New game` — сразу запускает новую партию с теми же настройками, без возврата на welcome screen.

Возврат на welcome screen происходит только если игрок открыл `Settings & Rules` и нажал в нём `Save & Close`.

### Confirm-попап для `Give up?`

При нажатии `Give up?` появляется модальное окно подтверждения (стиль и поведение — по `SERIES_STYLE_GUIDE.md` §6.2):

```text
title:   "Give up?"
body:    "You'll see one possible path, but it won't count as a win."
buttons: "Let me think again..." | "Yes, sorry!"
```

До закрытия модального окна состояние игры не меняется. Если игрок подтверждает (`Yes, sorry!`) — игра переходит в `finished` с исходом «Сдался» и показом одного валидного маршрута. Если выбирает `Let me think again...` — игра продолжается с того же места.

## Визуальные элементы

### Доска

- Квадратная.
- Центрируется в основной игровой зоне.
- Использует тёмно-бронзовые и светло-пергаментные клетки.
- Не использует чистую чёрно-белую шахматную палитру.
- Должна выглядеть как пол комнаты из книги, а не как стандартная chess board widget.

### Конь

- Готовый растровый спрайт `knight.webp`, лежит в папке проекта.
- Не Unicode-символ и не программная отрисовка — именно картинка.
- Должен выглядеть как живой шахматный конь из главы.
- Размер на доске: примерно `0.80 * cellSize`, центрирован в клетке, пропорции спрайта сохраняются.
- Текущая клетка под конём **дополнительно не подсвечивается** — спрайт сам по себе достаточно тёмный и хорошо читается на любых клетках доски.
- В режиме ожидания (пока игрок не сделал ход) конь дышит — лёгкая синусоидальная анимация масштаба около `1.0`, период ~2 с.

### Маршрут

- Посещённые клетки получают номер шага.
- Предыдущие клетки могут быть слегка затемнены или отмечены следом.
- Текущая клетка не получает дополнительной подсветки — её обозначает только спрайт коня.
- В режиме hints ON: допустимые следующие клетки подсвечиваются мягким бирюзовым мерцанием (того же ритма, что пульсация золотой кнопки `Start game`).
- В режиме hints ON: клетки-тупики (`0`) подсвечиваются красным мерцанием вместо бирюзового.
- В режиме hints OFF: допустимые клетки **не подсвечиваются** — доска чистая.

Все цвета — константами в объекте `COLOR` в `render.js`. Конкретные значения, оттенки клеток, бирюзовый цвет подсветки и параметры мерцания задаются в `UI_SPEC.md`.

## Структура состояния игры

Минимальная структура состояния:

```js
{
  phase: "welcome" | "playing" | "finished",

  // Settings (persisted in localStorage)
  boardSize: 5,
  showCounts: true,          // toggle for Warnsdorff continuation-count hints

  // Playing state (cleared at the start of each new game)
  knightPos: { x, y },
  startPos: { x, y },        // always (0, 0) in V1
  path: [{ x, y }],          // ordered list of visited cells, [0] is start
  visited: Set,              // same data as `path`, for O(1) membership checks
  availableMoves: [],        // valid next moves for current knightPos

  // Modal / result state
  modal: null | "confirmGiveUp",
  result: null,              // null while playing; outcome object on finished

  // Reserved for future extensions, not used in V1
  selectedHistoryStep: null, // selected step in optional move-history list; null in V1
}
```

### Что сохраняется между запусками

В `localStorage` сохраняются настройки:

| Ключ | Значение |
|------|----------|
| `knightPath_boardSize` | `5` / `6` / `7` / `8` |
| `knightPath_showCounts` | `"on"` / `"off"` |

Если в `localStorage` ничего не записано (первый запуск или очищенные данные), используются значения по умолчанию:

| Настройка | Default |
|-----------|---------|
| `boardSize` | `5` |
| `showCounts` | `on` |

Настройки записываются в момент нажатия `Save & Close` на экране настроек.

Прогресс текущей партии **не сохраняется**: при перезагрузке iframe маршрут теряется, игра возвращается на welcome screen.

Хранилище оборачивается в `safeStorage` с fallback (см. `SERIES_STYLE_GUIDE.md` §9).

### Поля, зарезервированные для расширений

В state остаётся одно поле, которое в V1 не управляется:

- `selectedHistoryStep` всегда `null`. В будущем — индекс выбранного хода в правой колонке-истории, к которому применяется откат по кнопке `Undo to selected`.

## Основной игровой цикл

### Welcome phase

1. Показывается приветственный экран с изображением и кнопками `Settings & Rules` / `Start game`.
2. По `Start game` создаётся новая партия с текущим `boardSize`, конь ставится в `(0, 0)`, фаза переключается в `playing`.

### Playing phase

1. Игрок видит доску и коня.
2. Допустимые клетки подсвечены, на каждой — число продолжений (Warnsdorff).
3. Клетки с нулём подсвечиваются красным.
4. Игрок кликает по допустимой клетке → выполняется ход:
   - обновляется `path` и `visited`;
   - `knightPos` переходит на новую клетку;
   - пересчитываются `availableMoves` и continuation counts;
   - проигрывается звук хода;
   - проверяется победа (все клетки посещены) или тупик (`availableMoves` пуст и есть непосещённые клетки).
5. Клик по недопустимой клетке: ход не выполняется, проигрывается негативный звук.
6. Кнопка `Undo` откатывает один последний ход (полный пересчёт состояния).
7. Кнопка `Give up?` открывает confirm-попап. До закрытия попапа состояние не меняется.
8. Игра продолжается до победы, тупика или подтверждённой сдачи — после чего фаза переключается в `finished`.

### Finished phase

1. Маршрут игрока остаётся на доске.
2. Если игрок сдался — поверх рисуется один валидный маршрут.
3. Показывается результат (см. «Итоги игры»).
4. Доступны `Settings & Rules` и `New game`.

## Рекомендованный scope для V1

Обязательно:
- доска 5×5–8×8 (стартовая клетка `(0, 0)`);
- клик по допустимым клеткам;
- негативный звук на клик по недопустимой клетке;
- подсветка допустимых ходов;
- настройка `Continuation hints: On / Off`;
- при `On` — бирюзовое мерцание на допустимых клетках, числа продолжений (Warnsdorff) поверх него, красное мерцание на нулях;
- при `Off` — чистая доска: допустимые клетки никак не выделены, чисел и красной подсветки нет;
- простой `Undo` на один ход;
- победа при полном покрытии доски;
- `Give up?` с confirm-попапом;
- показ одного валидного маршрута после сдачи;
- settings (модалка, открывается с welcome и finished);
- welcome screen с `knight-path-cover.webp`;
- финальный экран с `Settings & Rules` и `New game`;
- сохранение `boardSize` и `showCounts` в localStorage (через `safeStorage`);
- debug `D` / `I`.

Можно отложить:
- полноценную историю ходов справа;
- откат к выбранному ходу (`Undo to selected`);
- случайный выбор стартовой клетки из набора валидных;
- дополнительные режимы сложности;
- обучающий onboarding.

### Открытый вопрос для V1: показ маршрута после сдачи

После подтверждения `Give up?` нужно показать один валидный Knight's Tour. Возможные подходы:

1. **Генератор на лету** — backtracking с эвристикой Варнсдорфа из стартовой клетки `(0, 0)`. На досках 5×5–8×8 работает быстро (порядок миллисекунд), без UI-фриза.
2. **Фиксированные решения** — заранее записанный валидный маршрут для каждого размера доски (4 жёстко закодированных массива). Просто, предсказуемо, не зависит от качества алгоритма.

Решить в момент реализации. Если backtracking стабильно укладывается в один кадр — берём его (гибче, поддерживает будущий случайный старт). Если есть проблемы со стабильностью или временем — фиксированные маршруты как fallback.

## Важный дизайн-принцип

Эта игра не должна выглядеть как обычный шахматный тренажёр.

Она должна ощущаться как испытание из комнаты Башни:
- холодный клетчатый пол;
- живой шахматный конь;
- Glow-подсветка возможных ходов;
- стратегия через числа продолжений.

Игра должна объяснять мысль главы через действие: свобода выбора не всегда ведёт к успеху; иногда сначала нужно пройти узкие места.
